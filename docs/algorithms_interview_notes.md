# 8 种深度学习算法面试讲解稿

## 1. CNN 卷积神经网络

**核心一句话：** CNN 用卷积核扫描图像，从局部纹理逐层抽象到高级语义。

**整体流程：** 输入图像 → 卷积 → ReLU → 池化 → 多层特征提取 → 分类/检测。

**分层结构：**
- 浅层：边缘、纹理、角点。
- 中层：局部形状、部件。
- 深层：目标语义。

**关键机制：** 局部感受野、权重共享、平移等变性、池化降维。

**易错点：** CNN 不是只做分类，YOLO、分割、密度估计也大量依赖 CNN Backbone。

**记忆口诀：** 卷积看局部，池化降尺寸，层层抽语义。

**自测问题：**
1. 为什么卷积比全连接更适合图像？
2. ReLU 有什么作用？
3. 池化层为什么能降低计算量？

## 2. YOLO 目标检测

**核心一句话：** YOLO 一次前向推理同时预测目标位置、类别和置信度。

**整体流程：** 图像 → Backbone → Neck → Head → 候选框 → NMS → 最终检测框。

**分层结构：**
- Backbone：提取图像特征。
- Neck：融合多尺度特征。
- Head：输出 box、objectness、class。

**关键机制：** 一阶段检测、多尺度预测、NMS 去重、置信度筛选。

**易错点：** YOLO 适合稀疏或中等密度目标；密集人群用框计数容易漏检，需要密度图方法辅助。

**记忆口诀：** 一眼看全图，多尺度找目标，NMS 去重框。

**自测问题：**
1. 一阶段检测和两阶段检测有什么区别？
2. NMS 为什么必要？
3. YOLO 为什么适合实时视频？

## 3. Density Map 人群密度估计

**核心一句话：** 密度图不逐个框人，而是预测人群分布，对密度图求和得到人数。

**整体流程：** 人头点标注 → 高斯核扩散 → 训练密度图 → 模型预测 → 积分人数 → 热力图展示。

**分层结构：**
- 输入图像：无人机/监控画面。
- 标签生成：Head Point → Gaussian Density Map。
- 网络预测：CNN / Dilated Conv / CAN / CSRNet 思路。
- 后处理：区域积分、热力图、告警。

**关键机制：** Head Point 标注、Gaussian Kernel、Density Map 积分、多尺度感受野。

**易错点：** 人群密度估计不是目标检测；密集区域不要求每个人都被完整检测出来。

**记忆口诀：** 点变热图，热图求和，密集不框人。

**自测问题：**
1. 为什么密集人群不用 YOLO 直接数人？
2. Density Map 怎么得到人数？
3. 无人机高度变化对密度估计有什么影响？

## 4. RNN / LSTM 循环神经网络

**核心一句话：** RNN 用隐藏状态记住过去，LSTM 用门控机制解决长期记忆问题。

**整体流程：** 序列输入 → 每个时间步更新隐藏状态 → 输出序列/最终状态。

**分层结构：**
- RNN Cell：输入 x_t、上一状态 h_{t-1}、输出 h_t。
- LSTM Cell：遗忘门、输入门、候选记忆、输出门、Cell State。

**关键机制：** 时间展开、隐藏状态传递、门控记忆、梯度传播。

**易错点：** RNN 不是“有记忆就一定好”，长序列会梯度消失；LSTM 是为了解决长期依赖。

**记忆口诀：** RNN 记过去，LSTM 加门控。

**自测问题：**
1. RNN 为什么能处理序列？
2. LSTM 的三个门分别干什么？
3. 为什么 Transformer 后来替代了很多 RNN 场景？

## 5. Transformer

**核心一句话：** Transformer 用 Self-Attention 让每个 token 直接关注其他 token，适合并行和长距离依赖。

**整体流程：** Token → Embedding → Position Encoding → Q/K/V → Attention → FFN → 多层堆叠。

**分层结构：**
- Embedding：把 token 变成向量。
- Self-Attention：计算 token 间关系。
- Multi-Head：并行关注多种关系。
- FFN：逐位置非线性变换。
- Residual + LayerNorm：稳定训练。

**关键机制：** Q/K/V、Softmax 权重、多头注意力、位置编码、残差连接。

**易错点：** Attention 不是简单相似度检索，而是用 QK 得到权重，再对 V 加权求和。

**记忆口诀：** Q 问问题，K 做匹配，V 给答案。

**自测问题：**
1. Q、K、V 分别是什么？
2. 为什么要除以 sqrt(d_k)？
3. Multi-Head Attention 的意义是什么？

## 6. AutoEncoder 自编码器

**核心一句话：** AutoEncoder 把输入压缩成低维表示，再尽量重建原始输入。

**整体流程：** 输入 x → Encoder → 隐变量 z → Decoder → 重建 x' → 计算重建误差。

**分层结构：**
- Encoder：降维压缩。
- Latent Code：低维语义表示。
- Decoder：从压缩表示重建。
- Loss：重建误差。

**关键机制：** 信息瓶颈、降维、重建损失、异常样本重建误差大。

**易错点：** AutoEncoder 不是分类模型，核心目标是重建输入。

**记忆口诀：** 先压缩，再还原，误差看异常。

**自测问题：**
1. AutoEncoder 为什么能用于异常检测？
2. 隐变量 z 的作用是什么？
3. AutoEncoder 和 PCA 有什么相似点？

## 7. GAN 生成对抗网络

**核心一句话：** GAN 让生成器造假、判别器鉴假，通过对抗训练生成逼真数据。

**整体流程：** 噪声 z → Generator → 假样本 → Discriminator → 真/假判断 → 对抗优化。

**分层结构：**
- Generator：从噪声生成数据。
- Discriminator：判断样本真假。
- Adversarial Loss：生成器和判别器相互博弈。

**关键机制：** min-max 对抗训练、真假样本分布逼近、判别反馈驱动生成器改进。

**易错点：** GAN 训练不稳定，可能模式崩溃；不是越训练越稳定。

**记忆口诀：** G 造假，D 鉴假，对抗变真。

**自测问题：**
1. GAN 里 G 和 D 分别优化什么？
2. 什么是模式崩溃？
3. GAN 和 Diffusion 的生成思路有什么不同？

## 8. Diffusion 扩散模型

**核心一句话：** Diffusion 先把数据逐步加噪成噪声，再学习一步步去噪生成数据。

**整体流程：** 正向加噪 → 学习预测噪声 → 反向去噪 → 生成样本。

**分层结构：**
- Forward Process：x_0 → x_t，逐步加噪。
- Denoising Network：预测噪声 ε。
- Reverse Process：x_t → x_{t-1}，逐步去噪。
- Sampling：从随机噪声生成数据。

**关键机制：** 高斯噪声调度、时间步 embedding、噪声预测、逐步采样。

**易错点：** Diffusion 不是一次生成结果，而是多步去噪；Stable Diffusion 在 latent space 中扩散。

**记忆口诀：** 先打乱，再复原，一步一步生成。

**自测问题：**
1. 正向扩散和反向扩散分别做什么？
2. 模型训练时通常预测什么？
3. 为什么 Diffusion 生成质量高但采样较慢？
